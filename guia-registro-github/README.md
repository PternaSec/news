# Cómo acceder a los apartados de Red Team y Blue Team

Para mantener un entorno seguro y profesional, los laboratorios avanzados y las secciones de **Red Team** y **Blue Team** de PternaSec requieren autenticación. Afortunadamente, hemos integrado un sistema de registro rápido utilizando tu cuenta de GitHub.

En esta guía te explicamos cómo vincular tu cuenta y desbloquear todo el potencial de la plataforma.

## Paso 1: Iniciar el proceso de autenticación

1. Dirígete a la página principal de la plataforma o al **Dashboard**.
2. En la parte superior derecha, encontrarás el botón **Entrar** (con el ícono de GitHub).
3. Haz clic en el botón. Esto iniciará el flujo de autenticación segura utilizando OAuth.

## Paso 2: Autorizar la aplicación en GitHub

Si no tienes una sesión activa de GitHub en tu navegador, se te pedirá que ingreses tus credenciales (usuario y contraseña). 

Una vez que hayas iniciado sesión en GitHub (o si ya lo estabas):
1. Verás una pantalla de autorización de **PternaSec**.
2. Revisa los permisos solicitados. Únicamente necesitamos acceso a tu perfil público para identificarte (tu nombre de usuario y tu avatar).
3. Haz clic en el botón verde **Authorize** para permitir el acceso.

## Paso 3: Explorar Red Team y Blue Team

Una vez completada la autorización, serás redirigido de vuelta al Dashboard de PternaSec de forma automática.

¡Felicidades! Tu cuenta ya está validada. Ahora notarás que:
- Las restricciones de las categorías avanzadas han desaparecido.
- Puedes navegar libremente por el apartado de **Red Team** para explorar técnicas de ataque, exploits y herramientas ofensivas.
- Tienes acceso completo al apartado de **Blue Team**, donde encontrarás guías de mitigación, análisis de logs y estrategias de defensa.

### ¿Por qué pedimos registro con GitHub?

Utilizar GitHub nos permite verificar que los usuarios pertenecen a la comunidad de desarrolladores e investigadores de ciberseguridad, evitando el abuso automatizado de nuestros laboratorios (bots) y fomentando un ecosistema de aprendizaje colaborativo y ético.

*Si tienes algún problema durante el registro, no dudes en abrir un issue en nuestro repositorio principal.*
